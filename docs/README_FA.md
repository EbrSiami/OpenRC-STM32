<a name="transmitter"></a>
## 🎮 فرستنده (Transmitter)

فریم‌ور فرستنده رادیوکنترل ۸ کاناله برای میکروکنترلر STM32F103. 
این فریم‌ور یک **حلقه کنترلی ۵۰۰ هرتز** را به همراه سیستم منوی کامل روی نمایشگر OLED اجرا می‌کند و برای انجام تنظیمات هیچ نیازی به کامپیوتر ندارد.

### قابلیت‌ها

**کنترل و ورودی‌ها**
- ۸ کانال: ۴ کانال آنالوگ گیمبال‌ها + ۲ پتانسیومتر آنالوگ + ۲ سوئیچ دیجیتال
- تریم‌های دیجیتال برای Roll، Pitch و Yaw — ذخیره در حافظه EEPROM با قابلیت قفل در مرکز (center-snap) و محدودیت فرکانسی لبه‌ها
- حالت‌های قابل انتخاب برای کانال تراتل (گاز): حالت هواپیما (صفر در نیمه پایینی استیک) یا حالت معمولی

**پردازش سیگنال**
- فیلتر نویز میانگین متحرک نمایی (EMA) روی هر ۶ ورودی آنالوگ
- راهنمای گام‌به‌گام کالیبراسیون استیک‌ها (تنظیم مرکز، حداقل و حداکثر)
- امکان تنظیم باند مرده (Deadband)، ضریب اکسپو (Expo از ۱۰۰-٪ تا ۱۰۰+٪) و دوآل ریت (Dual Rate از ۱۰ تا ۱۰۰٪)
- تنظیمات Sub-Trim و EPA (تنظیم نقاط انتهایی) برای ۳ کانال اصلی
- امکان معکوس‌سازی جهت هر کانال به صورت مجزا (برای هر ۸ کانال)
- ۵ حالت پیش‌فرض میکسر: معمولی (Normal)، حالت V-Tail A/B و Delta A/B

**رابط کاربری OLED (نمایشگر 0.96 اینچی)**
- داشبورد آنی: نمایش ولتاژ باتری، تایمر پرواز، وضعیت D/R و نشانگر اتصال فرستنده (TX)
- نمایش زنده وضعیت کانال‌ها به صورت نمودار میله‌ای (کانال‌های ۱-۴ و ۵-۸)
- تایمر پرواز: شمارش معکوس یا کرونومتر، فعال‌سازی با بالا بردن تراتل، به همراه هشدارهای صوتی بازر در زمان‌های ۱ دقیقه، ۳۰ ثانیه و ۱۰ ثانیه آخر پرواز
- سیستم منوی کامل: تنظیمات Expo، Dual Rate، معکوس‌سازی کانال، میکسر، کالیبراسیون، EPA/Sub-Trim و حالت شبیه‌ساز (Simulator Mode)
- صفحه خوش‌آمدگویی متحرک هنگام روشن شدن دستگاه

**قابلیت اطمینان و امنیت**
- معماری غیرمسدودکننده (Non-blocking) — عدم استفاده از تابع `delay()` در حلقه اصلی برنامه
- استفاده از ماژول NRF24L01+ در سرعت ۲۵۰ کیلوبیت بر ثانیه، حداکثر توان خروجی و غیرفعال بودن پکت‌های تأیید خودکار (auto-ACK) — نرخ به‌روزرسانی ۵۰۰ هرتز
- سیستم بازر اولویت‌بندی‌شده (۱۴ الگوی صوتی) — هشدارهای با اولویت بالا بر تمامی تنظیمات جاری ارجحیت دارند
- مدیریت حافظه EEPROM همراه با مقدار جادویی (Magic Number) و CRC؛ بازنشانی خودکار به تنظیمات پیش‌فرض امن در صورت خرابی داده‌ها
- پایشگر باتری‌های لیتیوم‌پلیمر ۲ سلولی و ۳ سلولی (2S/3S) به همراه هشدار کد مورس SOS در صورت کاهش ولتاژ

### ⚠️ حجم حافظه فلش — بسیار مهم

حجم فریم‌ور کامپایل‌شده تقریباً **۷۰ کیلوبایت** است. میکروکنترلر استاندارد STM32F103C**8**T6 (برد معروف به Blue Pill با ۶۴ کیلوبایت حافظه فلش) **پشتیبانی نمی‌شود**. لطفاً از یکی از گزینه‌های زیر استفاده کنید:

- STM32F103C**B**T6 (۱۲۸ کیلوبایت حافظه فلش) — گزینه پیشنهادی
- کلون‌های چینی STM32F103 با ۱۲۸ الی ۲۵۶ کیلوبایت فلش (رایج هستند و معمولاً بدون مشکل کار می‌کنند)
- برد STM32 Black Pill (با اعمال تغییرات در پایه‌ها)

### ترتیب پایه‌ها (Pinout)

| قطعه | پایه STM32 | توضیحات |
| :--- | :--- | :--- |
| **NRF24L01+** | | |
| CE | `PB8` | |
| CSN | `PB9` | |
| SCK / MISO / MOSI | `PA5 / PA6 / PA7` | SPI1 |
| **نمایشگر OLED** | | |
| SDA / SCL | `PB7 / PB6` | I2C1 |
| **گیمبال‌ها** | `PA0 – PA3` | ADC |
| **پتانسیومترهای کمکی** | `PB0, PB1` | ADC |
| **سوئیچ‌های کمکی** | `PB4, PB5` | ورودی دیجیتال |
| **دکمه‌های تریم** | `PB15, PA8–PA10, PA15, PB3` | فعال در سطح پایین (Active low) |
| **دکمه‌های ناوبری منو** | `PB12 – PB14` | بالا / پایین / تایید (Enter) |
| **بازر (Buzzer)** | `PC13` | فعال در سطح بالا (Active high) |
| **سنجش ولتاژ باتری** | `PA4` | تقسیم ولتاژ مقاومتی |

### نکات سخت‌افزاری

**منبع تغذیه ماژول NRF24L01+**
این ماژول به نویز منبع تغذیه بسیار حساس است. یک خازن ۱۰ الی ۱۰۰ میکروفاراد را مستقیماً روی پایه‌های VCC و GND ماژول لحیم کنید. استفاده از یک رگولاتور مجزای AMS1117-3.3 نسبت به خروجی ۳.۳ ولت روی خود برد Blue Pill اولویت دارد، زیرا رگولاتور روی برد ممکن است توانایی تأمین جریان لحظه‌ای مورد نیاز ماژول را نداشته باشد.

**مقاومت‌های پول‌آپ I2C**
مقاومت‌های پول‌آپ داخلی STM32 برای ارتباط پایدار I2C ضعیف هستند. دو مقاومت خارجی ۴.۷ کیلواهم روی هر دو پایه SDA و SCL قرار دهید.

**راه‌انداز بازر (Buzzer Driver)**
بازر را مستقیماً به پایه GPIO میکروکنترلر متصل نکنید. از یک ترانزیستور NPN (مانند 2N2222) یا یک ماسفت کوچک به عنوان مدار راه‌انداز استفاده کنید.

**تقسیم مقاومتی ولتاژ باتری**
مقادیر پیشنهادی: R1 = 22 kΩ (به قطب مثبت باتری)، R2 = 6.8 kΩ (به زمین/GND). حداکثر ولتاژ قابل اندازه‌گیری: حدود ۱۴ ولت (ایمن برای باتری‌های حداکثر ۳ سلولی).

### ساخت و برنامه‌ریزی (Build & Flash)

**پیش‌نیازها:** نرم‌افزار VS Code با افزونه PlatformIO و پروگرمر ST-Link V2.

```bash
git clone https://github.com/EbrSiami/OpenRC-STM32.git
```

پوشه `transmitter/` را در PlatformIO باز کنید، با کلیک روی نماد **✓** پروژه را کامپایل کرده و با نماد **→** آپلود کنید.

همچنین می‌توانید فایل پیش‌ساخته `.elf` را از صفحه Releases و با استفاده از نرم‌افزار STM32CubeProgrammer روی میکروکنترلر فلش کنید (استفاده از فرمت `.elf` به دلیل آدرس‌دهی خودکار، نسبت به فرمت `.bin` اولویت دارد).

## 📡 گیرنده (Receiver)

فریم‌ور گیرنده جمع‌وجور ۸ کاناله. پکت‌های دریافتی از فرستنده را از طریق ماژول NRF24L01+ رمزگشایی کرده و به صورت همزمان، سیگنال‌های مجزای **PWM** و خروجی تایمر سخت‌افزاری **PPM** را ارائه می‌دهد. این فریم‌ور دارای سیستم قطع خودکار ایمن (Failsafe) ۵۰۰ میلی‌ثانیه‌ای است که در صورت قطع ارتباط رادیویی، گاز را قطع می‌کند.

**کامپایل با فلگ `-Os`** (بهینه‌سازی برای حجم حافظه) انجام شود؛ زیرا میکروکنترلر G030F6P6 تنها دارای ۳۲ کیلوبایت حافظه فلش و ۸ کیلوبایت حافظه رم (RAM) است.

### وضعیت نشانگر LED

| وضعیت ال‌ای‌دی | مفهوم |
| :--- | :--- |
| روشن ثابت | ارتباط برقرار است — پکت‌ها با موفقیت دریافت می‌شوند |
| چشمک‌زن آرام (۵۰۰ میلی‌ثانیه) | در انتظار دریافت سیگنال از فرستنده |
| چشمک‌زن سریع (۱۰۰ میلی‌ثانیه) | خطای سخت‌افزاری ماژول NRF24L01+ |

### مقادیر حالت ایمن (Failsafe)

در صورت قطع ارتباط رادیویی بیش از ۵۰۰ میلی‌ثانیه، مقادیر تمامی کانال‌ها به حالت پیش‌فرضِ امن تغییر می‌کنند: رول / پیچ / یاو روی ۱۵۰۰ میکروثانیه (حالت خنثی)، تراتل روی ۹۰۰ میکروثانیه (خاموش شدن موتورها) و سایر کانال‌های کمکی (Aux) روی ۱۰۰۰ میکروثانیه قرار می‌گیرند.

### ترتیب پایه‌ها (Pinout)

| قطعه | پایه STM32 | توضیحات |
| :--- | :--- | :--- |
| **NRF24L01+** | | |
| CE | `PA4` | |
| CSN | `PA5` | |
| SCK / MISO / MOSI | `PA1 / PA6 / PA2` | SPI |
| **خروجی‌های PWM** | | |
| CH1 Roll | `PA0` | |
| CH2 Pitch | `PA3` | |
| CH3 Throttle | `PA7` | |
| CH4 Yaw | `PB0` | |
| CH5 Aux1 | `PA11` | |
| CH6 Aux2 | `PA12` | |
| CH7 Aux3 | `PB3` | |
| CH8 Aux4 | `PB7` | |
| **خروجی PPM** | `PC14` | تایمر سخت‌افزاری TIM3 |
| **ال‌ای‌دی وضعیت** | `PC15` | |

### نکات سخت‌افزاری

**منبع تغذیه ماژول NRF24L01+**
همانند بخش فرستنده — یک خازن ۱۰ الی ۱۰۰ میکروفاراد را مستقیماً روی پایه‌های VCC و GND ماژول قرار دهید.

**پایه‌های RST و BOOT0**
این پایه‌ها نباید به صورت شناور (رها) رها شوند. پایه BOOT0 را به زمین (GND) متصل کنید (برای حالت کارکرد عادی). پایه RST نیز باید دارای یک مقاومت پول‌آپ به VCC باشد (معمولاً ۱۰ کیلواهم).

**عدم نیاز به کریستال خارجی**
میکروکنترلر G030F6P6 از اسیلاتور داخلی RC خود استفاده می‌کند و نیازی به کریستال خارجی ندارد.

**خطوط سیگنال PWM / PPM**
یک مقاومت سری حدوداً ۱ کیلواهم روی هر یک از پایه‌های خروجی PWM و PPM قرار دهید تا از میکروکنترلر محافظت شده و اثر نویز (Ringing) در کابل‌های بلند کاهش یابد.

### ساخت و برنامه‌ریزی (Build & Flash)

فایل `receiver/receiver.ino` را در Arduino IDE باز کنید. در منو، `build_flags` را روی `-Os` تنظیم کنید.

یا با استفاده از ST-Link V2 و نرم‌افزار STM32CubeProgrammer آن را فلش کنید (از فایل `.elf` در بخش Releases استفاده کنید).

## 🖥️ شبیه‌ساز - ایستگاه زمینی رادیوکنترل (Simulator)

یک برنامه تحت ویندوز که داده‌های دریافتی از **حالت شبیه‌ساز (Simulator Mode)** فرستنده را از طریق درایور جوی‌استیک مجازی **vJoy** به انواع شبیه‌سازهای پرواز (مثلاً شبیه‌سازهای FPV) منتقل می‌کند.

### نحوه کارکرد

فعال کردن حالت شبیه‌ساز در فرستنده، ارتباط رادیویی ماژول NRF24L01+ را غیرفعال کرده و داده‌های کانال‌ها را از طریق پورت بومی USB CDC میکروکنترلر STM32 (پورت سریال مجازی) به کامپیوتر ارسال می‌کند.

> **نکته:** این بخش از یک پروتکل سریال اختصاصی استفاده می‌کند و مبتنی بر پروتکل‌های iBUS یا SBUS نیست. زمان‌بندی استاندارد iBUS روی درگاه USB CDC میکروکنترلر STM32 (که رفتار متفاوتی نسبت به تراشه‌های اختصاصی USB-UART مانند CH340 دارد) پایداری کافی را نداشت؛ به همین دلیل یک پروتکل اختصاصی برای مدیریت کامل فریم‌ها و تشخیص خطا توسعه داده شد. این پروتکل شامل یک **بیت کنترل خطای CRC-8** روی تک‌تک پکت‌ها است.

حالت شبیه‌ساز **هیچ‌گاه در EEPROM ذخیره نمی‌شود**؛ فرستنده همیشه در حالت رادیوی فعال روشن خواهد شد. این موضوع یک تصمیم طراحی برای حفظ ایمنی پرواز است.

### مراحل فعال‌سازی — گام‌به‌گام

1. فرستنده را روشن کنید (به طور پیش‌فرض، رادیو فعال است).
2. در منوی OLED به بخش **Features** بروید.
3. گزینه **Simulator Mode** (آخرین گزینه، پیش‌فرض: `Off`) را انتخاب کرده و آن را روی `On` تنظیم کنید.
4. فرستنده به طور خودکار ماژول NRF24L01+ را خاموش کرده و شروع به ارسال داده‌ها از طریق پورت USB می‌کند.
5. فرستنده را از طریق کابل USB به کامپیوتر متصل کنید.
6. برنامه **RC Ground Station** را باز کرده، پورت COM مربوطه را انتخاب کنید (پورت‌های متصل به STM32 با علامت ✓ مشخص شده‌اند) و روی دکمه **Connect Device** کلیک کنید.
7. نمودارهای میله‌ای کانال‌ها به صورت آنی شروع به حرکت می‌کنند. اکنون فرستنده شما به عنوان جوی‌استیک شناسایی شده است.

**قبل از اتصال:** حتماً نرم‌افزار vJoy و درایور پورت سریال مجازی STM32 (Virtual COM Port) را نصب کنید. هر دوی این موارد در فایل نصبی ایستگاه زمینی گنجانده شده‌اند یا می‌توانید از طریق دکمه **⚡ Install Required Drivers** در داخل نرم‌افزار آن‌ها را نصب کنید.

### نگاشت کانال‌ها (Channel Mapping)

| کانال رادیویی | محور جوی‌استیک vJoy | محدوده مقادیر |
| :--- | :--- | :--- |
| CH1 Roll | X | 0–32767 |
| CH2 Pitch | Y | 0–32767 |
| CH3 Throttle | Z | 0–32767 |
| CH4 Yaw | Rx | 0–32767 |
| CH5 AUX1 | Ry | 0–32767 |
| CH6 AUX2 | Rz | 0–32767 |
| CH7 AUX3 | Button 1 | 0 / 1 |
| CH8 AUX4 | Button 2 | 0 / 1 |

## 📦 فایل‌های انتشار (Releases)

هر نسخه منتشر شده شامل فایل‌های زیر است:

| فایل | توضیحات |
| :--- | :--- |
| `transmitter_vX.X.X.elf` | فریم‌ور فرستنده — جهت فلش با STM32CubeProgrammer |
| `transmitter_vX.X.X.bin` | فریم‌ور فرستنده — فایل باینری خام |
| `receiver_vX.X.X.elf` | فریم‌ور گیرنده — جهت فلش با STM32CubeProgrammer |
| `receiver_vX.X.X.bin` | فریم‌ور گیرنده — فایل باینری خام |
| `RC_Ground_Station_vX.X.X.exe` | برنامه شبیه‌ساز تحت ویندوز به همراه پکیج کامل درایورهای vJoy و STM32 |

> **پیشنهاد می‌شود:** از فایل‌های با پسوند `.elf` همراه با نرم‌افزار STM32CubeProgrammer استفاده کنید. فایل‌های ELF شامل متادیتای آدرس حافظه فلش هستند که مانع از نوشتن اشتباه داده‌ها روی نواحی غیرمجاز حافظه می‌شود.

<a name="transmitter"></a>
## 🎮 Передатчик

Прошивка 8-канального RC-передатчика для STM32F103. Работает на **цикле управления 500 Гц** с полноценной системой меню на OLED-экране — для настройки ПК не требуется.

### Функционал

**Управление и ввод**
- 8 каналов: 4 аналоговых стика + 2 аналоговых потенциометра + 2 цифровых переключателя
- Цифровые триммеры для Roll (крен), Pitch (тангаж) и Yaw (рыскание) — сохраняются в EEPROM с фиксацией по центру и крайними лимитами
- Выбираемый режим газа: самолетный (ноль в нижней половине) или обычный

**Обработка сигналов**
- Экспоненциальный фильтр шума (EMA) на всех 6 аналоговых входах
- Пошаговый мастер калибровки стиков (центр, минимум, максимум)
- Настраиваемая мертвая зона (deadband), экспонента (Expo от −100% до +100%), двойные расходы (Dual Rate от 10% до 100%)
- Sub-Trim и EPA (регулировка конечных точек) для 3 основных каналов
- Инверсия каналов (для всех 8 каналов по отдельности)
- 5 пресетов микшера: Normal, V-Tail A/B, Delta A/B

**Интерфейс OLED (0.96")**
- Панель управления в реальном времени: напряжение батареи, таймер полета, статус D/R, индикатор связи TX
- Графическое отображение каналов в реальном времени (CH1–4 и CH5–8)
- Таймер полета: обратный отсчет или секундомер, запускаемый газом, со звуковыми оповещениями на 1 мин / 30 сек / последних 10 сек
- Полноценная система меню: Expo, Dual Rate, инверсия каналов, микшер, калибровка, EPA/Sub-Trim, режим симулятора
- Анимированная заставка при запуске

**Надежность и безопасность**
- Неблокирующая архитектура — без использования `delay()` в основном цикле
- NRF24L01+ на скорости 250 кбит/с, максимальная мощность, автоподтверждение (auto-ACK) отключено — частота обновления 500 Гц
- Приоритетный движок зуммера (14 паттернов) — высокоприоритетные сигналы тревоги переопределяют все настройки
- EEPROM с контрольной суммой Magic Number + CRC; автосброс к безопасным настройкам по умолчанию при повреждении данных
- Мониторинг батареи 2S/3S LiPo с SOS-сигнализацией при низком напряжении

### ⚠️ Объем флэш-памяти — Важно

Размер скомпилированной прошивки составляет около **70 КБ**. Стандартная плата STM32F103C**8**T6 (Blue Pill, 64 КБ Flash) **не поддерживается**. Используйте один из следующих вариантов:

- STM32F103C**B**T6 (128 КБ Flash) — рекомендуется
- Китайские клоны STM32F103 с флэш-памятью 128–256 КБ (распространены, обычно работают)
- STM32 Black Pill (с корректировкой распиновки)

### Распиновка (Pinout)

| Компонент | Вывод STM32 | Примечания |
| :--- | :--- | :--- |
| **NRF24L01+** | | |
| CE | `PB8` | |
| CSN | `PB9` | |
| SCK / MISO / MOSI | `PA5 / PA6 / PA7` | SPI1 |
| **OLED-дисплей** | | |
| SDA / SCL | `PB7 / PB6` | I2C1 |
| **Стики (джойстики)** | `PA0 – PA3` | ADC |
| **Доп. потенциометры** | `PB0, PB1` | ADC |
| **Доп. переключатели** | `PB4, PB5` | Цифровой ввод |
| **Кнопки триммирования** | `PB15, PA8–PA10, PA15, PB3` | Активный низкий уровень |
| **Кнопки навигации** | `PB12 – PB14` | Вверх / Вниз / Ввод (Enter) |
| **Зуммер (Buzzer)** | `PC13` | Активный высокий уровень |
| **Датчик батареи** | `PA4` | Делитель напряжения |

### Примечания по аппаратному обеспечению

**Питание NRF24L01+**
Модуль чувствителен к помехам по питанию. Припаяйте конденсатор емкостью 10–100 мкФ непосредственно к выводам VCC и GND модуля. Выделенный стабилизатор AMS1117-3.3 предпочтительнее встроенной шины 3.3 В платы Blue Pill, которая может не обеспечивать достаточный пиковый ток.

**Подтягивающие резисторы I2C**
Внутренние подтягивающие резисторы STM32 слишком слабы для стабильного I2C. Добавьте внешние резисторы 4.7 кОм на линии SDA и SCL.

**Драйвер зуммера**
Не подключайте зуммер напрямую к выводу GPIO. Используйте транзистор NPN (например, 2N2222) или небольшой MOSFET в качестве схемы драйвера.

**Делитель напряжения батареи**
R1 = 22 кОм (к плюсу батареи), R2 = 6.8 кОм (к GND). Максимальное измеряемое напряжение: ~14 В (безопасно для 3S LiPo).

<a name="receiver"></a>
## 📡 Приемник

Компактная прошивка 8-канального приемника. Декодирует пакеты NRF24L01+ от передатчика и одновременно выводит как отдельные сигналы **PWM**, так и аппаратный сигнал **PPM**. Включает отказоустойчивый режим (failsafe) на 500 мс, который отключает газ при потере связи.

**Компилируйте с флагом `-Os`** (оптимизация по размеру) — STM32G030F6P6 имеет всего 32 КБ Flash и 8 КБ RAM.

### Статус светодиода

| Режим | Значение |
| :--- | :--- |
| Горит непрерывно | Связь активна — пакеты принимаются |
| Редкое мигание (500 мс) | Ожидание сигнала |
| Частое мигание (100 мс) | Аппаратная ошибка NRF24L01+ |

### Значения аварийного режима (Failsafe)

При потере связи (> 500 мс) все каналы устанавливаются в безопасные значения по умолчанию: Roll / Pitch / Yaw → 1500 мкс (нейтраль), Throttle → 900 мкс (моторы отключены), все каналы Aux → 1000 мкс.

### Распиновка (Pinout)

| Компонент | Вывод STM32 | Примечания |
| :--- | :--- | :--- |
| **NRF24L01+** | | |
| CE | `PA4` | |
| CSN | `PA5` | |
| SCK / MISO / MOSI | `PA1 / PA6 / PA2` | SPI |
| **Выходы PWM** | | |
| CH1 Roll | `PA0` | |
| CH2 Pitch | `PA3` | |
| CH3 Throttle | `PA7` | |
| CH4 Yaw | `PB0` | |
| CH5 Aux1 | `PA11` | |
| CH6 Aux2 | `PA12` | |
| CH7 Aux3 | `PB3` | |
| CH8 Aux4 | `PB7` | |
| **Выход PPM** | `PC14` | Аппаратный таймер TIM3 |
| **Светодиод состояния** | `PC15` | |

### Примечания по аппаратному обеспечению

**Питание NRF24L01+**
Аналогично передатчику — добавьте конденсатор емкостью 10–100 мкФ непосредственно между выводами VCC и GND модуля.

**Выводы RST и BOOT0**
Они не должны оставаться в «плавающем» состоянии. Подтяните BOOT0 к GND (для нормального режима работы). Вывод RST должен иметь подтягивающий резистор к VCC (обычно 10 кОм).

**Внешний кварцевый резонатор не требуется**
G030F6P6 использует внутренний встроенный RC-генератор. Внешний кварц не требуется.

**Линии сигналов PWM / PPM**
Добавьте последовательный резистор номиналом около 1 кОм на каждый выходной вывод PWM и PPM для защиты микроконтроллера и уменьшения звона (ringing) на длинных кабелях.

### Сборка и прошивка

Откройте файл `receiver/receiver.ino` в Arduino IDE. Установите `build_flags в -Os` в меню.

Или прошейте с помощью ST-Link V2 через STM32CubeProgrammer (используйте файл `.elf` из релизов).

<a name="simulator"></a>
## 🖥️ Симулятор (Наземная станция RC)

Приложение для Windows, которое связывает **режим симулятора (Simulator Mode)** передатчика с любым симулятором FPV через виртуальный джойстик **vJoy**.

### Принцип работы

Режим симулятора передатчика отключает радиомодуль NRF24L01+ и вместо этого отправляет данные каналов через встроенный USB CDC (виртуальный COM-порт) контроллера STM32.

> **Примечание:** Здесь используется собственный последовательный протокол, а не iBUS или SBUS. Стандартные тайминги iBUS работали нестабильно через встроенный USB CDC STM32 (который ведет себя иначе, чем выделенные чипы USB-UART типа CH340), поэтому для полного контроля над структурой фреймов и обнаружением ошибок был разработан кастомный протокол. Протокол включает контрольную сумму **CRC-8** для каждого пакета.

Режим симулятора **никогда не сохраняется в EEPROM** — передатчик всегда загружается с активным радиомодулем. Это сделано намеренно в целях безопасности полетов.

### Активация — пошагово

1. Включите передатчик (радиомодуль активен по умолчанию).
2. Перейдите в раздел **Features** в меню на OLED-экране.
3. Выберите **Simulator Mode** (последний пункт, по умолчанию: `Off`) и установите значение `On`.
4. Радиомодуль NRF24L01+ автоматически отключится, и передатчик перейдет в режим вывода данных по USB.
5. Подключите передатчик к ПК через USB-кабель.
6. Откройте **RC Ground Station**, выберите соответствующий COM-порт (порты STM32 отмечены символом ✓) и нажмите **Connect Device**.
7. Графики каналов начнут обновляться в реальном времени. Передатчик теперь работает как джойстик.

**Перед подключением:** установите программу vJoy и драйвер виртуального COM-порта STM32. Оба компонента включены в установщик наземной станции, либо их можно открыть с помощью кнопки **⚡ Install Required Drivers** внутри приложения.

### Карта каналов (Channel Mapping)

| Канал радиоуправления | Ось vJoy | Диапазон |
| :--- | :--- | :--- |
| CH1 Roll | X | 0–32767 |
| CH2 Pitch | Y | 0–32767 |
| CH3 Throttle | Z | 0–32767 |
| CH4 Yaw | Rx | 0–32767 |
| CH5 AUX1 | Ry | 0–32767 |
| CH6 AUX2 | Rz | 0–32767 |
| CH7 AUX3 | Кнопка 1 | 0 / 1 |
| CH8 AUX4 | Кнопка 2 | 0 / 1 |

```bash

python simulator/simulator.py
```

## 📦 Релизы

Каждый релиз включает в себя:

| Файл | Описание |
| :--- | :--- |
| `transmitter_vX.X.X.elf` | Прошивка передатчика — прошивать через STM32CubeProgrammer |
